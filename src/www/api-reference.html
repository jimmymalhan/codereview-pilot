API Reference

Complete reference for Claude Debug Copilot REST API. All endpoints require POST/GET requests with JSON payloads.

Core Endpoints

POST
/api/diagnose

Submit a single incident for diagnosis

Request Body:

{ "incident": "Database query takes 45 seconds to complete" }

Response:

{ "id": "diag-1773121469588", "incident": "...", "result": { "router": { ... }, "retriever": { ... }, "skeptic": { ... }, "verifier": { ... } }, "timestamp": "2026-03-10T...", "status": "completed" }

✓ 200 OK - Diagnosis created successfully

✗ 400 Bad Request - Invalid incident (< 10 chars or > 2000 chars)

✗ 429 Too Many Requests - Rate limit exceeded (100/hour)

POST
/api/batch-diagnose

Process multiple incidents in one request (up to 100)

Request Body:

{ "incidents": [ "Issue 1: Memory leak in cache", "Issue 2: Slow query performance", "Issue 3: Connection timeout" ] }

✓ 200 OK - All incidents processed

✗ 400 Bad Request - Empty array or > 100 items

GET
/api/diagnose/:id

Retrieve a previously created diagnosis by ID

Example:

GET /api/diagnose/diag-1773121469588

✓ 200 OK - Diagnosis found and returned

✗ 404 Not Found - Diagnosis with ID not found

GET
/api/diagnostics?page=1&limit=20

List all diagnoses (paginated)

page: Page number (default: 1)

limit: Results per page (default: 20, max: 100)

✓ 200 OK - List of diagnostics returned

GET
/api/diagnose/:id/export?format=json

Export diagnosis in JSON or CSV format

format: "json" or "csv" (default: json)

✓ 200 OK - File exported in requested format

✗ 404 Not Found - Diagnosis not found

GET
/api/analytics

Get dashboard metrics and statistics

Returns:

{ "totalDiagnoses": 5, "diagnosesLast24h": 2, "averageConfidence": 0.94, "byStatus": { "completed": 5 }, "bySeverity": { "high": 3, "medium": 2 }, "mttrMinutes": 45, "successRate": 0.94 }
GET
/api/audit-log?limit=100

View immutable event trail (for compliance)

limit: Number of events (default: 100, max: 1000)

✓ 200 OK - Audit events returned with trace IDs

POST
/api/webhooks

Register a webhook for notifications

Request Body:

{ "url": "https://example.com/webhook" }

✓ 200 OK - Webhook registered

GET
/health

Check server health and uptime

Returns:

{ "status": "healthy", "timestamp": "2026-03-10T...", "uptime": 3600.5, "diagnostics": 5 }

✓ 200 OK - Server is healthy

Quick Examples

1. Submit a Diagnosis

curl -X POST http://localhost:3000/api/diagnose \ -H "Content-Type: application/json" \ -d '{ "incident": "Database connection pool exhausted" }'

2. Batch Process

curl -X POST http://localhost:3000/api/batch-diagnose \ -H "Content-Type: application/json" \ -d '{ "incidents": [ "Issue 1", "Issue 2", "Issue 3" ] }'

3. Get Analytics

curl http://localhost:3000/api/analytics

4. Export Results

curl http://localhost:3000/api/diagnose/diag-123/export?format=json

Status Codes

200 OK

Request succeeded. Response contains the requested data.

400 Bad Request

Invalid request. Check your parameters and request body.

Common causes: Missing fields, invalid format, text too short/long

404 Not Found

Resource not found. The diagnosis ID doesn't exist.

429 Too Many Requests

Rate limit exceeded. Maximum 100 requests per hour per IP.

Retry after the "retryAfter" seconds specified in response.

500 Internal Server Error

Server error occurred. Try again later. Include the trace ID in support requests.
