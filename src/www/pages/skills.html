Skills Demonstrations

Test Evidence Verifier, Hallucination Detector, and Confidence Scorer with live data

1. Evidence Verifier

Validates that file:line citations actually exist in the repository. Prevents citing non-existent files or lines.

Use file paths relative to repository root. Example: src/run.js:1

2. Hallucination Detector

Detects AI hallucinations by checking if referenced fields and APIs actually exist in the schema.

Known schema fields: user.id, user.email, user.name, order.id, order.total, order.status

Known APIs: GET /api/users, POST /api/orders, GET /api/orders/:id

Check if a schema field exists. Try: user.email (valid) or user.phone (invalid)
Check if an API endpoint exists. Try: GET /api/users (valid) or DELETE /api/admin (invalid)

3. Confidence Scorer

Calculates confidence using formula: base + evidence(0.25) - hallucination(0.35) - contradiction(0.20)

Initial confidence before adjustments. Minimum 0.70 required for approval.
