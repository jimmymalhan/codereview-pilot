Evidence-First Debugging System

Diagnose recurring backend failures using evidence, not guesses

+===========================================================================+
|          5-AGENT PIPELINE ARCHITECTURE                                    |
+===========================================================================+
|                                                                           |
|  [INCIDENT] --> Router --> Retriever --> Skeptic --> Verifier --> Critic  |
|                   |          |            |           |           |       |
|                Classify    Gather      Challenge   Validate    Approve    |
|                 Type       Evidence     Theory     Evidence      Gate     |
|                                                                           |
|  Output Contract:                                                         |
|  * Root Cause        * Evidence Citations    * Tests                      |
|  * Fix Plan          * Rollback Plan         * Confidence                 |
|                                                                           |

Project Goals

  1. Diagnose recurring backend failures using evidence first
  2. Produce complete output contracts: root cause, evidence, fix plan, rollback, tests, confidence
  3. Never invent fields, tables, APIs, regions, or files - retrieve before explaining
  4. Use verifier to block unsupported nouns and skeptic for materially different theories
  5. Ensure all edits are approved before deployment with confidence gates (>= 0.70)
