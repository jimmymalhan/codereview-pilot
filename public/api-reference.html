🔌 API Endpoints

POST /api/diagnose

Submit incident for real-time diagnosis

Request:

{ "incident": "Database connection pool exhausted at 2 AM" }

Response (Success):

{ "incident": "...", "router": "Classification output", "retriever": "Evidence gathered", "skeptic": "Alternative theory", "verifier": "Final diagnosis with fix plan" }

Response (No Credits): 402 status with link to add credits

Response Structure

Field Type Description
incident string Original incident description
router string Failure classification (schema drift, write conflict, etc.)
retriever string Exact evidence with file:line citations
skeptic string Competing theory from different failure family
verifier string Final verified diagnosis (root cause, evidence, fix, rollback, tests, confidence)

📜 Diagnosis Output Contract

Every diagnosis includes:

Component Format Example
Root Cause Text (backed by evidence) "Pool size 50 insufficient for peak load"
Evidence Array of citations ["src/pool.js:45", "logs/api:15:30", "metrics: 50/50"]
Fix Plan Step-by-step instructions "1. Increase DB_POOL_SIZE to 200\n2. Add release hook\n3. Deploy"
Rollback Reversal procedure "Set DB_POOL_SIZE=50, restart servers"
Tests Validation cases ["Load test 500 users", "Monitor metrics"]
Confidence 0.0 - 1.0 score 0.92

🔄 4-Agent Pipeline Details

Stage 1: Router - Classify

Input: Your incident description

Output: Failure classification with reasoning

  • Identifies failure family (schema drift, write conflict, stale read, bad deploy, auth, dependency)
  • Analyzes temporal patterns and impact scope
  • Sets foundation for evidence gathering

Stage 2: Retriever - Gather Evidence

Input: Incident + router classification

Output: Exact evidence with file:line citations

  • Pulls code references (src/file.js:line)
  • Extracts log timestamps and snippets
  • Gathers schema definitions
  • No guessing—only verified facts

Stage 3: Skeptic - Challenge

Input: Incident + router + retriever

Output: Alternative diagnosis from different failure family

  • Generates competing theory
  • Questions primary diagnosis
  • Forces rigor through healthy skepticism

Stage 4: Verifier - Validate

Input: All prior outputs

Output: Final verified diagnosis

  • Rejects unsupported assertions
  • Validates claims against evidence
  • Produces output contract with confidence score
  • Human approval gate before deployment

⚙️ Non-Negotiable Rules

1. Never invent fields, tables, APIs, regions, or files
Everything must be retrieved or provided by user
2. Retrieve before explaining
No guessing or inference without evidence
3. Verifier blocks unsupported nouns
Every claim must cite specific evidence
4. Skeptic produces materially different theory
Not just shade on first answer—genuine alternative
5. No edits until plan is approved
All deployments require human gate
